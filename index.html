Visualization: Recorded Earthquakes, March 31.
Project One, Julia.

Each circle, colored according to a severity on the richter scale, represents a unique instance. The blue lines indicate present day tectonic plates boundaries. Close proximity to the tectonic plate boundaries greatly increases the likelihood and frequency of earthquakes.
In designing this map, I used the CartoDB chloropleth setting, which represents each earthquake instance with a colored circle based on the magnitude of the instance. When you hover over each circle, the magnitude and location appears. If you click on a circle, the magnitude, latitude, longitude, and location appears. I added a picture behind the location, which is the same picture for each entity. I did figure out how to have the specific google street view picture appear (by copying the script from a blog post found online), but since it was missing for most of the locations, I didn't use this. I thought an alternative could be to write a script that pulls the first image of a google image search on the specific place, but this was much more complicated than I am qualified for!

The legend in the bottom right hand corner explains the range of values associated with each of the different colors. However, I didn't use the CartoDB settings and instead fiddled with HTML code to change the size of the bullets and corresponding text, make some of the text italicized, change the text color, and add a sub-header. I spent a while trying to add a background image in the legend--I wanted to include the richter logarithmic curve--but while I eventually figured out how to do so, it looked more distracting than helpful and so I didn't include it. If I had more expertise, I think it would have been cool to make a basic logarithmic curve in R, and then plot the ranges--by color--on this as a replacement for the text in the legend.

I wanted to experiment with adding layers, so I spent quite a while searching for a new dataset to overlay with the initial chloropleth map. I hoped to find one that was relevant to earthquake frequency and magnitude, as well as increased understanding of the topic. I first experimented with a dataset that mapped fault lines. However, I only found data for the U.S., so I kept looking. Eventually, I found the plate tectonic boundaries dataset. The source provided the data in three separate files, which I merged into a single data table in CartoDB. I added this new table as a single layer to my map, choosing the color blue for the boundary lines, to contrast with the chloropleth layer. I would have liked to add an entry in the legend for the tectonic plate boundaries, with a blue line as the symbol. However, I could not figure out how to change the bullet point to a line, so I left it out.

Source, earthquake data: CartoDB. Source, tectonic plate boundaries: http://www.ig.utexas.edu/research/projects/plates/pt.info.htm.

Visualization: Renewable Energy in the U.S.
Project One, Julia.

Using the bubble technique, this map illustrates the number of renewable energy power plants registered with the U.S. Energy Information Administration in each state in 2013. The bubbles sit upon a chlorpleth layer, which is colored according to the average amount of energy (kwh per capita) used by the residents of each state in 2010.When you zoom into the map, the green bubbles disappear and instead the individual power plants, colored by energy source, appear. Interestingly, states with lower average energy use per capita also appear to have clusters of renewable energy power plants. While certainly not indicating a causal link, there may be some connection between lower energy use and adoption of renewable energy sources.

Because this visualization has a large number of colorful data points, I used a gray scale for the chloropleth layer to create contrast. I used green for the bubbles simply because it green is usually associated with environmentally-conscious products.The bigger the green bubble, the more renewable energy in the state. The individual power plants, are depicted with bright inidividual points, which also contrast nicely with the gray-scale chloropleth. They only appear on zoom 5.

I spent quite a while changing the CSS code associated with the legend and info-windows. I wanted to have the legend correlate with the zoom function so that the boxe depicting the number of power plants disappeared when the green bubbles disappear and the energy source legend only appears when they also appear on the map. While I could not figure out how to do this, I did make the boxes much smaller, change the type face, format the text, and fix the symbols slightly. I also used CSS to format the info window associated with each layer.

The power plant data is provided by the U.S. Energy Information Administration. However, it is split into a number of massive files. I ultimately used the data from the power plants and pulled the type of energy from the generator file. This assumes that the generators all operate on the same source of power. In Google Refine, I transformed the type of energy from abbreviations to categories of energy, grouping points like the anthracite coal, bituminous coal, lignite coal, coal-derived synthesis gas, subbituminous coal, and refined coal into one generic category of coal.Ultimately all of this data overwhelmed the graph and so I decided just to focus on renewable energy power plants. The number of renewable energy power plants by state is actually from the same U.S. EIA data, but I found the sums for each state, matched it with state boundary geographic data, and then made a new data table and layer. The energy use by state statistics stem from a different chart, which I joined with geographic data in CartoDB.

I would have liked to calculate the average energy produced by each power plant. However, each power plant is associated with a number of different generators. While I think this is possible, it would have required much more manipulation of the data in excel in order to find the average energy of each generator, the average of all generators associated with an individual plant, and then match that number to the unique plant identifier in a different excel file.
